# Dungeon exploded view

An exploded 3D viewer for stacked dungeon floors. Each floor is generated by the
real `xsofy.terrain` code and drawn as a tilted, isometric layer, so you can see
a whole multi-floor dungeon as a tower and inspect any level.

It's a single self-contained HTML file (CSS 3D transforms + 2D canvas — no
three.js, no build, no server).

## View it

- **Hosted:** https://raw.githack.com/mparrett/xsofy/tool/dungeon-3d-viewer/tools/dungeon-3d/index.html
- **Local:** open `index.html` in a browser. It loads the bundled `floors.js`
  (seed 42, 5 floors) via a `<script>` tag, so `file://` works — no server needed.

## Controls

- **Drag** to orbit, **scroll** to zoom, **shift/space-drag** to pan.
- **Click a floor** for a flat top-down view of just that level (floors above it
  hidden), like standing on it in-game. **Double-click** returns to the 3D stack.
- Sliders: explode gap, tilt, spin, zoom, cell aspect, floor thickness, dim level.
- Per floor: **show** toggles visibility, **dim** fades it so you can see floors
  beneath. Helpers: `all`, `top` (solo the top floor), `dim ↑` (dim everything
  above the deepest shown floor).
- **load export…** (or drop a file on the window) loads a different export —
  see below.

## Generating data

`export-floors.lg` dumps floors as the data the viewer reads. Run from the xsofy
root (needs the native-xxh3 `lg`):

```
lg tools/dungeon-3d/export-floors.lg            # window.FLOORS wrapper, seed 42, 5 floors
lg tools/dungeon-3d/export-floors.lg 64 8       # seed 64, 8 floors
lg tools/dungeon-3d/export-floors.lg json 64    # raw JSON, seed 64
```

Args are order-free: `json` selects raw JSON output; the first bare number is the
seed, the second is the floor count. The default `window.FLOORS = {…};` form is
what `index.html` bundles; the `json` form is the portable file you drop into the
viewer. The upload accepts either.

`floors-seed64.json` and `floors-seed7-d8.json` are sample exports to drop in.

## Data format

```
{ "w":79, "h":30, "seed":42, "floors":[
    { "depth":1, "down":[x,y], "up":[x,y], "grid":[<tile codes, row-major>] }, … ] }
```

`grid[x + y*w]` is a tile type (0–19). The viewer maps types to glyphs and colors
from a palette inlined from `xsofy/terrain.lg` (`tile-chars` / `tile-styles`).
`down`/`up` are the stair coordinates; they're exported but the current viewer
renders stairs straight from the grid and doesn't use them.
